# UI-UX-Experiment3

## Aim:
To improve the usability of the Zomato application by identifying delivery availability issues and redesigning the interface to display only nearby and deliverable restaurants using heuristic evaluation and A/B testing methods.

## Algorithm:
Start Select Applications – Choose 2–3 competitor apps/websites in the same domain (e.g., Swiggy, Zomato, Uber Eats). Define Scope – Decide which features to evaluate (search, checkout, order tracking, payment, help). Choose Evaluation Criteria – Use Nielsen’s 10 usability heuristics. Perform Evaluation – For each heuristic, check compliance/violations in each app. Document Issues – Record problems with severity rating (Low/Medium/High). Analyze Results – Compare strengths and weaknesses of apps.

## Design Changes Implemented in Version B

Location-first approach: Delivery address is confirmed at the top of the home screen before any
restaurants are shown. A "4 km delivery radius" indicator is displayed.

Search bar redesign: Rounded pill-style search bar with placeholder "Search restaurants, foods" —
cleaner and more inviting than the flat rectangle in Version A.

Filter chips — Near Me & Delivers here: Two prominent filter pills immediately below the search bar.

"Near Me" is active (red fill) by default. "Delivers here" is a toggle — solving the core problem.

Restaurant count feedback: "Showing 3 Restaurants" text gives immediate visibility of how many
options match — aligns with Heuristic #1.

Availability badge on each card: Every restaurant card shows either a green "Available" or grey "Not
Available" pill — user knows before clicking. This directly solves the core UX problem.

Consistent card layout: Each card shows: food image thumbnail, name, heart (wishlist), availability
badge, star rating, price (Rs.), delivery time — all in one scannable row.

Bottom navigation: Clean 4-icon nav bar: Home, Search, Wishlist (heart), Profile — minimal and
standard.
